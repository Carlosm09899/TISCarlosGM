Justificación del Diseño de la Base de Datos

Tabla Servicios

Propósito

Es la tabla principal, su propósito es almacenar toda la información específica y detallada de cada producto o servicio que ofreces.

Tabla Categorias

Propósito

El propósito de esta tabla es agrupar y clasificar los servicios de una manera ordenada y consistente.

Tabla Estados

Propósito

Esta tabla de catálogo sirve para definir una lista oficial de los posibles estatus que un servicio puede tener.

Tabla Tipos_Control_Stock

Propósito

El propósito de esta tabla de catálogo es definir los diferentes métodos para gestionar el inventario de los servicios o productos.

Normalización de la Base de Datos

Para asegurar que los datos estén bien organizados y sean consistentes, la base de datos se estructuró siguiendo las tres primeras formas normales (1FN, 2FN y 3FN).

1NF (Primera Forma Normal)

Cada tabla tiene una clave primaria única y todas las columnas son atómicas.

2NF (Segunda Forma Normal)

La tabla está en 1NF y todos los atributos no clave dependen completamente de la clave primaria.

3NF (Tercera Forma Normal)

La tabla está en 2NF y ningún atributo no clave depende de otro atributo no clave.

Código SQL para Crear Tablas

CREATE DATABASE SERVICIOS;

-- --- Tabla de Categorías ---
CREATE TABLE Categorias (
    ID_Categoria INT IDENTITY(1,1) PRIMARY KEY,
    Nombre_Categoria VARCHAR(100) NOT NULL UNIQUE,
    Descripcion_Categoria TEXT
);

-- --- Tabla de Estados ---
CREATE TABLE Estados (
    ID_Estado INT IDENTITY(1,1) PRIMARY KEY,
    Nombre_Estado VARCHAR(50) NOT NULL UNIQUE
);

-- --- Tabla de Tipos de Control de Stock ---
CREATE TABLE Tipos_Control_Stock (
    ID_Tipo_Control INT IDENTITY(1,1) PRIMARY KEY,
    Nombre_Control VARCHAR(100) NOT NULL UNIQUE
);

-- --- Tabla Principal de Servicios ---
CREATE TABLE Servicios ( 
    ID_Servicio INT IDENTITY(1,1) PRIMARY KEY,
    Nombre_Servicio VARCHAR(200) NOT NULL,
    Descripcion_Detallada TEXT,
    Precio_Venta DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
    -- (resto de los campos...)
    
    -- Claves Foráneas que conectan con las tablas catálogo
    ID_Categoria INT,
    ID_Estado INT,
    ID_Tipo_Control INT,
    
    FOREIGN KEY (ID_Categoria) REFERENCES Categorias(ID_Categoria),
    FOREIGN KEY (ID_Estado) REFERENCES Estados(ID_Estado),
    FOREIGN KEY (ID_Tipo_Control) REFERENCES Tipos_Control_Stock(ID_Tipo_Control)
);
